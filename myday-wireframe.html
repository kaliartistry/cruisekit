Where Does the Data Come From?

TODAY TAB

1

Ship Time & Port Time

Calculated automatically from the sailing itinerary the user already entered in the Plan pillar. We know the port, we know the timezone.

Auto from itinerary
2

All-Aboard Countdown

Pulled from the sailing's itinerary data. Each port day has a departure time. We calculate the countdown in real-time.

Auto from itinerary
3

Daily Schedule Events

User adds their own events. Dinner reservations, show times, spa appointments, excursion departures. Quick-add with presets or custom entry. Each event gets an optional push notification.

User enters manually
4

"What's Open Now" Strip

Ship venues with typical operating hours, bundled per-ship in our database. Approximate hours based on cruise line patterns (e.g., buffet 6am-9:30pm, pool bar 10am-6pm). Users can correct/update.

Bundled ship data User-correctable

SPEND TAB

5

Budget Baseline

Seeded from the Plan pillar's True Cost Calculator. If the user estimated $4,200 total, that becomes the budget target. Categories (drinks, dining, spa, etc.) carry over too.

Auto from Plan calculator
6

Charge Logging

User taps to log each charge. Quick-add presets have cruise-line-specific default prices (e.g., Carnival cocktail = $13, Royal = $15). One tap adds, long-press to edit amount. Custom entry for anything else.

User taps to log
7

Budget vs Actual

Calculated in real-time. Compares logged spend against Plan estimates. Shows daily burn rate and projects end-of-cruise total. "Plan said $800 for drinks, you're at $340 on day 3."

Auto-calculated

KEY INSIGHT

No cruise line API needed. The Today tab is a personal planner (like Google Calendar for your cruise). The Spend tab is a personal ledger (like a budgeting app). Both are user-input driven with smart defaults. This means: works fully offline, zero partnership dependency, and ships on day one.
